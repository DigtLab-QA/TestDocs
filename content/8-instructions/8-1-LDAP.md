LDAP

Провайдер LDAP предназначен для авторизации пользователя через настроенный Active Directory.
Что такое Active Directory?

Создание LDAP-провайдера в сервисе

Шаг 1. Создание нового профиля

1.1 В форме авторизации, вызываемой по адресу https://id.trusted.plus/ нажать кнопку “Создать аккаунт”, в появившейся форме заполнить поля “Имя”, “Телефон или адрес электронной почты” (указать e-mail), а также заполнить два поля для создания нового пароля и его подтверждения. Проставить чекбокс с согласием на обработку персональных данных и кликнуть кнопку “Создать аккаунт”.

1.2 Ввести код подтверждения, высланный на указанный при регистрации email

1.3 Авторизоваться под учетными данными из подшага 1.1

Примечание: создать новый аккаунт можно  также посредством любого из провайдеров, представленных в нижней части окна авторизации


Шаг 2. Создание api-приложения

2.1 Перейти на вкладку “Мои приложения”, нажать иконку добавления нового приложения и выбрать тип приложения - API 

2.2 В открывшейся форме указать название приложения, прописать url сайта и url контроллера авторизации, а также указать пароль и его подтверждение. Нажать кнопку “Создать”.




Шаг 3. Создание LDAP-провайдера

3.1 Перейти на вкладку “Мои провайдеры”, нажать иконку добавления нового провайдера  и выбрать тип приложения - LDAP 

3.2 В открывшейся форме указать название LDAP-провайдера, заполнить адрес сервера LDAP (поле “ldap_url”), контекст учетных записей LDAP (поле “ldap_base”), имя домена LDAP (поле “ldap_domain”). При необходимости заполнить поля фильтра поиска учетной записи и списка сопоставления атрибутов LDAP (“ldap_filter” и “ldap_mapping”), а также поля описания провайдера и адреса профиля пользователя. Нажать кнопку “Создать”.




Шаг 4. Добавление созданного LDAP-провайдера в приложение

4.1 Перейти на вкладку “Мои приложения” и выбрать приложение из шага 2

4.2 В области “Список провайдеров” нажать кнопку добавления нового 
провайдера 

4.3 В открывшемся модальном окне проставить чекбокс напротив провайдера, созданного на шаге 3 и нажать кнопку “Сохранить”









Список провайдеров приложения при этом обновиться и в нем отразится вновь добавленный провайдер:



Шаг 5. Добавление аккаунта LDAP-провайдера в профили пользователя

5.1 Перейти на вкладку “Мой профиль” и внешний аккаунт LDAP

После успешного добавления аккаунта пользователю доступна авторизация через провайдер LDAP.

Добавить аккаунт LDAP к профилю пользователя можно иным способом

5.2 Откройте виджет авторизации и выберите провайдер LDAP для авторизации

5.3 Введи свои данные подключения
5.4 Выберите “Привязать аккаунт”

5.5 Введите логин и пароль существующего аккаунта


Шаг 6. Регистрация пользователя через аккаунта LDAP-провайдера 

6.1 Откройте виджет сервиса и выберите авторизацию через провайдер LDAP
6.2 Выберите “Создать аккаунт”

6.3 Заполните необходимые данные для регистрации

Шаг 7. Смена пароля аккаунта LDAP-провайдера 
7.1 Для изменения пароля аккаунта LDAP необходимо:

Почта учетной записи AD должна быть привязана к аккаунту пользователя Trusted;
Аккаунт AD привязан к профилю пользователя через провайдер LDAP в профиле пользователя Trusted
7.2 Порядок изменения пароля:
Открыть форму авторизации через провайдер LDAP

Нажать кнопку “Сменить пароль”. Откроется форма ввода адреса электронной почты

Ввести адрес электронной почты, которая привязана к аккаунту AD и профилю пользователя Trusted
Подтвердите почту отправленных на указанный адрес кодом
Введите новый пароль
